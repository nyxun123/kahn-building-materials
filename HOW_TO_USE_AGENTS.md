# 🚀 Agent团队使用指南（小白版）

## 📖 什么是Agent团队？

Agent团队就像您的专业开发团队，每个Agent都是某个领域的专家。您只需要告诉他们要做什么，他们会自动协作完成任务。

## 🎯 如何开始使用？

### 步骤1：选择合适的Agent
根据您要做的事情，选择对应的Agent：

| 我想做什么 | 找哪个Agent | 说什么 |
|-----------|------------|--------|
| 修改网站界面 | Frontend Lead Agent | "帮我修改首页的颜色搭配" |
| 添加新页面 | UI/UX Design Agent | "帮我设计一个新的产品展示页面" |
| 修复手机显示问题 | Mobile Responsive Agent | "手机上按钮太小了，帮我修复" |
| 添加新语言 | I18n Agent | "帮我添加法语支持" |
| 数据库问题 | Database Agent | "帮我添加一个用户评论功能的数据表" |
| 网站部署 | Deployment Agent | "帮我把网站发布到线上" |
| 代码检查 | Code Review Agent | "帮我检查代码有没有问题" |

### 步骤2：简单直接地提出需求
**好的例子：**
- "我想在首页添加一个产品轮播图"
- "手机上菜单打不开，帮我修复"
- "我想让网站支持俄语"
- "帮我添加用户留言功能"

**避免的例子：**
- "帮我做一个很炫酷的东西"（太模糊）
- "按照最新的技术标准实现..."（太技术化）

### 步骤3：让Agent自动协作
您不需要管理Agent之间的协作，他们会自动：
- 🤝 相互配合完成复杂任务
- 📋 更新任务进度
- ✅ 检查彼此的工作质量
- 🚀 最终一起部署

## 💡 实际使用场景

### 🎨 场景1：我想修改网站外观
**您说：** "首页看起来太单调了，帮我美化一下"

**Agent团队自动工作：**
1. UI/UX Design Agent 设计新的视觉方案
2. Frontend Lead Agent 实现代码修改
3. Mobile Responsive Agent 确保手机显示正常
4. Code Review Agent 检查代码质量
5. QA Testing Agent 测试功能
6. Deployment Agent 部署到线上

### 🔧 场景2：我想添加新功能
**您说：** "我想添加一个在线客服聊天功能"

**Agent团队自动工作：**
1. Project Lead Agent 分析需求
2. Database Agent 设计聊天数据结构
3. Backend Lead Agent 开发聊天API
4. Frontend Lead Agent 开发聊天界面
5. Security Agent 检查安全性
6. 所有Agent协作测试和部署

### 🐛 场景3：网站出问题了
**您说：** "网站打不开了，帮我修复"

**Agent团队自动工作：**
1. Monitoring Agent 诊断问题
2. 相关Agent修复问题
3. QA Testing Agent 验证修复
4. Deployment Agent 重新部署

## 🎭 在Claude.ai Projects中的具体操作

### 1. 项目设置
在您的Claude.ai Projects中：
- 把AGENTS.md上传到项目知识库
- 在项目说明中写明："请按照AGENTS.md中的角色分工协作"

### 2. 与Agent对话
**开始对话时：**
```
@Frontend Lead Agent 帮我修改首页样式
```
或者
```
我想添加新功能，请Project Lead Agent协调团队完成
```

### 3. 查看进度
Agent会自动告诉您：
- 🔄 正在做什么
- ✅ 完成了什么  
- ⏰ 下一步计划
- 🤝 需要其他Agent配合什么

## 🎯 最佳实践建议

### ✅ 这样做效果更好：
1. **具体明确**：说清楚您想要什么效果
2. **提供例子**：给Agent看看您喜欢的参考
3. **及时反馈**：Agent问您问题时及时回复
4. **信任团队**：让Agent自主协作，不要微管理

### ❌ 避免这样做：
1. 不要同时给多个Agent分配冲突的任务
2. 不要绕过Team Lead直接指挥Junior Agent
3. 不要在Agent工作时频繁修改需求
4. 不要忽略Agent提出的技术建议

## 🆘 遇到问题怎么办？

### 常见问题解决：
| 问题 | 解决方法 |
|------|---------|
| Agent不理解我的需求 | 多提供一些具体例子和截图 |
| 不同Agent给出矛盾建议 | 让Project Lead Agent协调统一 |
| 任务进度慢 | 询问是否遇到技术难题，需要调整方案 |
| 结果不满意 | 具体说明哪里不满意，让相关Agent改进 |

### 🚨 紧急情况：
- **网站崩溃**：直接说"网站出问题了，请紧急修复"
- **安全问题**：说"发现安全漏洞，请Security Agent立即处理"
- **部署失败**：说"部署失败了，请DevOps团队解决"
